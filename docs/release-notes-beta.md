# Clawworker Enterprise · Beta 变更摘要与已知局限

> P7 收敛复核产出物(2026-07-20)。面向种子客户/公测用户与内部交付,说明这一版**修了什么、还差什么**。
> 原则:**能兑现的写清楚,不能兑现的也写清楚**。数据分析产品最贵的是信任,含糊其辞比承认局限伤害更大。

---

## 一、这一版做了什么

23 轮优化循环,修复 20+ 处缺陷,测试从 252 增至 **306(stub)/ 311(real 后端)**,双后端全绿。
其中**约一半属于"数字算错 / 显示错 / 语义反转"这类零容忍问题** —— 逐条列在下面,便于评估影响面。

### 1.1 口径与数字正确性(命门,占比最高)

| 场景 | 问题 | 影响 |
|---|---|---|
| 密态解密(全路径) | 同态把精确 0 解成 ~1e-15,击穿所有除零护栏 | 利润率炸成 **-5e19** 并混进排名表 |
| 定时任务解密 | 加密前 `fillna(0)` 且不还原 | 周转天数 NaN(不周转=最差)变成 **0 天(最快=最好),语义反转** |
| 多维透视 | mean/min 的"合计"把各列均值/极值**相加** | 华东"平均"370000,高于任何单列均值(真值 206667) |
| 多维透视 | 缺失组合对非加性聚合填 0 | 没卖过的品类显示"均价 0 / 最低 0" |
| 应收账龄 | 负账龄(未到期)/缺失账龄被 `pd.cut` 静默丢弃 | 账龄表与应收总账**对不上**(样例差 3 万) |
| 帕累托 ABC | 负值(退货冲减)计入分母 | **累计占比突破 100%** |
| 库存周转 | 数据异常行显示无意义负天数 | -60.8 天 / -1095 天 |
| 行级均值比率 | "订单数"与均值分母不一致(除零行被剔除) | 财务按"均值×订单数"反推**对不上账** |
| 完成率呈现 | 整数百分数(85)被当小数 | 渲染成 **8500%**(单行表、含合法 0 的列两种触发) |
| 排序呈现 | 负值/异常被 `na_position="first"` 顶到降序表首行 | 亏损 -30% 出现在"利润率降序"表第一行,读者默认那是最优 |
| 定时任务调度 | `day_of_month` 一律夹到 28 | 设"每月31号"跑月结,**实际28号跑**、UI 却显示31号,提前3天赶在结账前 |
| Excel 摄取 | 两级表头(年份跨列合并)选错表头行 | 列名变 `Unnamed:N`、真表头沦为数据行,**静默成功** |
| TOP-N | 截断处并列项被静默丢弃 | 同为 90 分只有 B 进 TOP2,C/D 无声出局 |

### 1.2 安全与合规

- **TLS + TOFU 指纹锁定**:主机/客户端全 HTTPS,首连锁定证书指纹,证书变更时给出双指纹并引导人工核对。
- **授权吊销闭环**:三个解密入口前置会话新鲜度门,过期即拦、须回主机重登(登录时应用吊销);
  离线吊销延迟上界 = 会话 TTL(默认 8h)。详见 `revocation-model.md`。
- **审计哈希链**:写入时崩溃留下的残行不再被误报为"疑似篡改",且不再吞掉后续事件;
  真篡改(改字段/删中间行)仍能检出。并发写入链完整。
- **沙盒加固**:AST 扫描 + 禁反射逃逸 + 禁文件写出 + exec 超时;CSRF/DNS-rebinding 防护;
  密钥 DPAPI 加密落盘 + Windows ACL 收紧;登录限速。

### 1.3 能力与呈现

- 新增 **人效(per_capita)** 技能(人均 = 总额÷去重人头,合计按加权口径)。
- **16 个技能全部配齐口径说明**,渲染在表顶供财务核对(此前 11 个缺失)。
- 图表分页按**实体块对齐**(不把某个实体的时间序列拦腰切到两张图),图表数封顶。
- 密文留存版与明文版**列结构一致**;超行数上限时截断并在表顶标注,明文版始终保留全量。

## 二、已知局限(明确不兑现的部分)

> 这一节是**承诺边界**。销售/售前请勿超出此范围表述。

### 2.1 架构固有,无法回避

1. **本地持钥 + 离线 = 无法密码学阻断解密。**
   私钥在客户端本地,解密是本机运算。一个已拿到密钥且离线的人,绕过本产品 UI 直接调 HE 库仍可解密自己手上的密文。
   吊销在本产品里是**策略 + 有界时间 + 审计**三层控制,不是实时密码学阻断。缓解:短会话 TTL、
   短授权有效期、吊销后密钥轮换、全程审计留痕。详见 `docs/revocation-model.md`。
2. **哈希链只防部分篡改。** 删掉日志尾部若干行后,剩余链仍自洽,本地校验查不出。
   抵御整档重写需主机侧定期锚定(尚未实现)。
3. **已导出的明文 Excel 不受吊销影响**,与任何文件分发一致。

### 2.2 能力尚未覆盖(roadmap)

| 项 | 现状 | 计划 |
|---|---|---|
| **跨密文文件 join** | 不支持。要求关联另一张表时,模型会明确拒绝并说明缺字段,不会编造 | GA 后补;当前建议单表分析 |
| **多级表头展平** | 两级表头不再损坏数据(能取到真表头行),但**年份等上层语义会丢失**(`收入` / `收入.1`) | GA 前补展平为 `2024年_收入` |
| **forecast 季节性** | 仅线性趋势外推,**不建模季节性/周期**;强季节性或有转折点的场景预测值仅供参考(已写入口径说明) | GA 后补 |
| **单行数据做分位** | 模型会诚实说明"1 行无法分位",但可能改用其它指标兜底(有说明) | 低优 |
| **cron 解析失败** | 创建时已校验表达式与依赖;仅"运行时 croniter 缺失"这一窄窗口会静默退化为每天,无告警日志 | 低优,补日志 |

### 2.3 验证边界(哪些结论是实测的,哪些不是)

- **实测过**:五大场景(销售/财务/库存/客户/HR)真实数据口径评审;`AGENT_BACKEND=real` 全量回归;
  四条主链路端到端(codegen / skill-plan / 密文导出往返 / 定时任务);真实 LLM(deepseek-v4-pro)
  验证提示词铁律确实生效;规模场景(200 实体 × 12 期)。
- **未实测**:多机部署(admin 与客户端异机)的证书信任流程仅本机验证过;
  百万行级数据仅有代码层护栏,未做真实压测;长时间(周/月级)连续运行的稳定性未验证。

## 三、诚实的自我评价

23 轮里有 **6 轮是"换一个没探过的面,就又挖出零容忍级问题"**,其中第 18 轮首次把分层测试串成端到端,
当场暴露"各层单测全绿、整体却算出 -5e19"。这说明:

> **收敛的准确含义是"已探过的面都干净了",不等于"不存在未知问题"。**

因此本版定位为 **Beta(公测)**,适合有人陪跑的种子客户,不建议直接用于无人复核的对外披露报表。
建议种子客户:① 前几次运行人工复核关键数字;② 优先跑已评审过的五大场景;
③ 遇到数字反常立刻反馈——每一个真实坏 case 的价值高于我们自己再造十个测试。
