# Clawworker Enterprise · 商用就绪路线图

> 把 `optimization-prompts.md` 的 P0–P7 每个环节,定出"做到什么程度才算商用可推广"的**验收线(Exit Gate)**,
> 并对照当前状态给出差距。判定分三档:✅达标 / 🟡接近(有明确小差距)/ ❌未达标(硬门槛)。
> 一句话原则:**这是数据分析产品,"数字算错"是命门;是隐私产品,"安全承诺兑现"是价值本身。这两条零容忍。**

---

## 一、每个环节的商用验收线 vs 现状

### P0 · 全景诊断(四维:场景覆盖/输出合理性/自修复/安全)
- **商用验收线**:连续 **两轮**四维诊断,**零 P0(致命)、零未修 P1(高危)**;所有发现要么修复,
  要么进入《已知可接受局限》清单(每条带缓解措施 + 触发条件)。
- **现状**:已做 **3 轮**诊断,P0/P1 全部修复并各自经独立对抗复核;测试 175→252。
- **差距**:剩余多为 P2 功能增强类;需**一次干净的最终诊断**确认无回归后即达标。
- **判定**:🟡 接近(再跑一轮确认即 ✅)

### P1 · 场景红队(刁钻输入)
- **商用验收线**:覆盖真实客户会打的 7 类边界;**安全越界 / 明文泄露 / 提示注入类必须全 ✅**;
  业务模糊类允许 🟡 但必须有"追问或显性声明假设"兜底;累计案例 ≥40;
  **且用真实 LLM 后端实测过所有"预期改善"项**(prompt 规则是 LLM 行为,静态验证不够)。
- **现状**:2 轮 27 例;安全类全 ✅;字段名投毒/条件分支/无意图乱码已修并加回归测试。
- **差距**:① 业务语义类 🟡 项只做了静态预测,**未用真实 LLM 跑过**;② 案例数需补到 ≥40。
- **判定**:🟡 接近(缺真实 LLM 验证 + 补案例)

### P2 · 系统提示词最小修复
- **商用验收线**:红队/诊断发现的问题都有对应**最小修复**(不顺手重构);系统提示词稳定
  (改一处不破坏他项);每处改动有**回归测试锁行为**(不锁措辞)。
- **现状**:大量最小修复已落地,均配回归测试(test_redteam_guardrails 等)。
- **判定**:✅ 达标

### P3 · Excel 输出合理性评审(财务总监视角)
- **商用验收线**:**5 大场景(销售/财务/库存/客户/HR)各出一份样表**,以挑剔财务总监视角评审,
  **无"当场吐槽"级问题**;尤其**数字口径零错误**(百分比/金额/加权/同比环比全对)。
- **现状**:真实数据实测了**销售场景**——时间排序、加权合计、亏损上色、除零、排名、口径 sheet 全部正确;
  发现一个 🟡 小瑕疵:LLM 生成代码把亏损行排到了降序表顶(排名列对,行位置观感差)。
- **差距**:❌ **财务/库存/客户/HR 四场景尚未系统出样表评审**;修亏损行排序瑕疵。
- **判定**:❌ 未达标(这是数据产品命门,必须补齐 5 场景评审)

### P4 · 自修复回环强化
- **商用验收线**:**5 类典型失败**(密态 API 误用/字段名错/类型不匹配/超出密态能力/AST 拦截)
  逐个验证——重试提示词信息量够 LLM 定位;有全局重试/时长硬上限;定时任务失败有熔断告警;
  修不回来的降级到固化技能或给明确可操作报错。
- **现状**:执行崩溃回喂 LLM(限次)、窗口未排序防缓存、定时任务熔断、全局 LLM 调用上限、
  exec 超时+可取消、DecryptionFailed 终态不瞎回退——机制齐全。
- **差距**:"5 类失败逐个跑真实 LLM 确认重试提示词够用"未系统做。
- **判定**:🟡 接近(机制到位,缺真实 LLM 逐类验证)

### P5 · 失败案例固化为回归测试
- **商用验收线**:所有已修案例转 pytest,**只测行为不测措辞**;CI 全绿;
  **`AGENT_BACKEND=real` 全量跑过**(stub 过 ≠ 密态真的过)。
- **现状**:252 测试全绿,含红队 guardrail;但**都是 stub 后端**,real 后端集成测试仅 5 个(密钥到位自动启用),未系统跑全量 real。
- **差距**:❌ **real 后端全量回归未跑**——密态实际行为(精度/近似/边界)未被测试覆盖。
- **判定**:❌ 未达标(real 后端全量必须过)

### P6 · 自动优化循环(/loop)
- **商用验收线**:挂 `/loop` 能自动"发现失败 → 最小修 → 固化 → 收敛";连续两轮无新失败 + 全绿=收敛。
- **现状**:未跑过自动循环。
- **差距**:属**持续改进机制**,商用**前不强制**;建议 GA 后开启作为运营手段。
- **判定**:🟡 非阻塞(上线后开)

### P7 · 收敛复核
- **商用验收线**:`AGENT_BACKEND=real` 全量 pytest 通过;提示词精简(删冗余/重复规则,越短越不顾此失彼);
  产出《变更摘要 + 已知局限》正式文档。
- **现状**:**已完成(第24轮,2026-07-20)**。`AGENT_BACKEND=real` 全量 311 绿;
  产出《Beta 变更摘要与已知局限》`docs/release-notes-beta.md`。
- **提示词精简结论:评估后决定不做形式化精简**。近似行分析显示重复项主要是分节标题(假阳性),
  真正重复的只有"规则+示例""规则+API清单"这类有意冗余;而现有提示词在第8/9/18/20轮已被真实 LLM
  验证生效(安全铁律/排序铁律/诚实拒绝/技能选型均正确)。无证据表明长度导致漏规则,
  为满足清单去删文本反而有回归已验证行为的风险 —— 待真实 LLM 出现漏规则证据时再定向精简。
- **判定**:✅ 达标(Beta 层面)

## 二、商用前"必须清零"的硬门槛(否则不能收费推广)

1. **数字口径零错误**——P3 五场景样表评审全部通过。数据产品算错一次=客户信任归零。
2. **real 后端全量回归通过**——P5/P7。密态精度、近似算子(max/min/分位数)、大表边界必须实测。
3. ~~**证书吊销的本地离线闭环**~~ —— **已闭环(第6轮)**:实现"短 TTL 强制回主机"——三个解密入口前置
   会话新鲜度门(`_session_fresh`),过期即拦截解密、须回主机重登(登录时应用吊销),离线吊销延迟上界=会话TTL(默认8h);
   并出《授权吊销模型与已知局限》(`docs/revocation-model.md`)诚实说明"本地持钥离线不可密码学阻断"+缓解(短TTL/密钥轮换/审计)。
4. **真实 LLM 验证红队 🟡 项不致命**——P1。确认 prompt 规则在真实模型上确实生效。

## 三、可"先上线、后迭代"的项(标进 roadmap,不阻塞首发)

- **跨密文文件 join**(最大功能缺口):首发先支持单表分析 + 明确提示"多表请分别上传";GA 后补。
- **forecast 季节性/多因子**:先线性回归 + 在 summary 声明"线性外推,强季节性场景仅供参考"。
- **Excel 多级表头/合并单元格解析**:先支持单表头 + 上传时提示。
- **P6 自动优化循环**:上线后作为运营机制开启。
- **admin 从异机远程访问的证书信任**:先要求 admin 在主机本机操作(localhost 已免警告)。

## 四、分层上市判断(用现状对照)

| 阶段 | 面向 | 前置门槛 | 现在能不能 |
|---|---|---|---|
| **内测 / 种子客户(Alpha)** | 强信任的少数客户,有人陪跑 | 核心能力跑通 + 安全护栏到位 + 单场景验证 | **✅ 现在就能**——已具备 |
| **公测(Beta)** | 更多客户,自助使用 | 补 P3 五场景评审 + real 后端全量 + 真实 LLM 验证红队 🟡 + 部署/多机文档 | 🟡 约 **1–2 周**补齐 |
| **正式 GA(收费推广)** | 公开市场 | Beta 全部 + 证书吊销闭环(或明确局限)+ 跨密文 join(或 roadmap 承诺)+ SLA 级自修复验证 + 定价/合同/支持体系 | ❌ 需 **额外 2–4 周** + 商务侧准备 |

## 五、建议的执行顺序(把差距变成任务)

1. **P3 五场景样表评审**(命门,最高优先)——财务/库存/客户/HR 各造真实数据跑一份,挑口径错误;顺手修亏损行排序瑕疵。
2. **P5/P7 real 后端全量回归**——用真实密钥跑 `AGENT_BACKEND=real pytest`,补密态边界测试。
3. **P1 真实 LLM 验证 🟡 项 + 补案例到 ≥40**。
4. **证书吊销本地闭环**决策:实现短 TTL,或写进《已知局限》。
5. **P7 收敛复核**:精简提示词 + 出《变更摘要 + 已知局限》正式文档。
6. GA 前:跨密文 join 的 roadmap 承诺、部署文档、定价/支持体系(商务侧)。

> 结论:**产品已跨过"能用、安全、单场景验证"的门槛,可进内测/种子客户。**
> 距**收费 GA** 的关键差距是三条零容忍门槛:五场景口径评审、real 后端全量、证书吊销闭环——
> 都是"确认与补测"性质,不是重造,预计 2–4 周可清零。
