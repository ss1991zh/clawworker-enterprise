# Clawworker 优化循环手册(可反复执行)

> 一个能**反复跑**的闭环:每跑一轮消化 1 个待办,跑干了就自动补货,直到某个里程碑收敛。
> 配套:`optimization-prompts.md`(每步的提示词)、`eval_cases.md`(红队案例)、`commercial-readiness.md`(验收线)。
> 铁律:**每个修复必配行为测试;每轮跑全量 pytest;涉及输出/安全的必须真实数据或 real 后端验证。**

---

## 一、单一待办源(BACKLOG)—— 所有要做的事只看这里

待办按优先级分 5 层,**永远从最高层的第一条取**。来源:红队 ❌/🟡、诊断发现、真实使用 bug、商用验收线差距。

**P级定义(取题顺序):**
- **T0 零容忍门槛**(算错数 / 明文泄露 / 崩溃挂死 / 安全绕过)—— 有一条就不能进下一里程碑
- **T1 高危**(误导性输出 / 自修复修不回 / 安全护栏缺口)
- **T2 场景覆盖**(宣传场景做不了 / 做不好)
- **T3 呈现质量**(Excel 观感 / 口径标注 / 图表)
- **T4 增强**(新能力 / roadmap 项)

## 二、每轮固定动作(THE LOOP · 一次消化一条)

```
1) 取题    从 BACKLOG 最高层取第一条未完成项。
2) 复现    用一条具体输入/一份真实数据复现问题;若是新问题 → 先追加进 eval_cases.md。
3) 定层    判断根因在哪一层:系统提示词 / 意图路由 / 技能代码 / 渲染 / 安全护栏 / 部署。
4) 最小修  只改必要的那一处,不顺手重构。
5) 验证    ① 全量 pytest 绿;
           ② 输出类 → 用真实数据跑一遍打开 Excel 查口径/格式;
           ③ 安全类 → 构造攻击输入确认被拦;
           ④ 能力类 → real 后端或真实 LLM 实测。
6) 固化    加一个"只测行为不测措辞"的回归测试(拒绝了/追问了/没泄露/口径对)。
7) 复核    口头复核 eval_cases 里前 5 个已通过案例,确认本次改动没让它们退化。
8) 记账    BACKLOG 打 [x];进度日志追加一行(日期·修了什么·pytest 数·通过率)。
9) 提交    git commit + push(一条待办一个 commit,便于回溯)。
```

**单轮止损**:若一条待办 3 次最小修仍不达标 → 拆小成子项重排 BACKLOG,或标"需真实 LLM/密钥环境",不死磕。

## 三、补货机制(BACKLOG 跑干时触发,保证循环不空转)

按顺序做,把新发现塞回 BACKLOG:
1. **红队一轮(P1)**:换新角度构造 12–15 个刁钻输入,预判现状,判定 ✅/🟡/❌,❌ 与高优 🟡 进 BACKLOG。
2. **场景评审一轮(P3)**:挑一个场景造真实数据出样表,财务总监视角挑刺,问题进 BACKLOG。
3. **真实使用回收**:把种子客户/自己用出来的坏 case 直接进 BACKLOG(优先级按 T0–T4 归类)。
4. 若**连续两轮补货都产不出新的 T0/T1**,且 pytest 全绿 → 当前里程碑判**收敛**,进下一里程碑。

---

## 四、收敛判据(每个里程碑一套 Exit Gate)

对照 `commercial-readiness.md`,里程碑收敛 = 该档 BACKLOG 的门槛项清零 + 连续两轮无新失败 + 全量绿:

- **Alpha 收敛**(种子客户):核心跑通 + 安全护栏 + ≥1 场景真实验证 —— **已达成**。
- **Beta 收敛**(公测):T0-1~T0-5 清零 + T1-1 完成 + 部署文档 + real 后端全量绿。
- **GA 收敛**(收费):Beta 全部 + T0-6(证书吊销)+ T2-1(跨密文 join 或 roadmap 承诺)+ SLA 级自修复验证。

每到一个里程碑收敛,跑一次 **P0 全景诊断 + P7 收敛复核**(real 全量 + 精简提示词 + 出《变更摘要+已知局限》)。

## 五、节奏建议(把循环挂到日历上)

| 频率 | 动作 |
|---|---|
| **每次坐下** | 跑 1–3 轮"每轮固定动作",消化 BACKLOG 顶部 |
| **每周** | 补货:红队 1 轮(P1)+ 场景评审 1 轮(P3) |
| **每个版本 / 里程碑** | P0 四维诊断 + P5/P7 real 后端全量 + 收敛判定 |
| **想全自动** | 用 `/loop` 挂 P6:自动"发现→修→固化→收敛",人只审 commit |
