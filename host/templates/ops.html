工作原理:开机自启只负责在登录时拉起 supervisor.py 一个守护进程; 由守护进程负责启动、健康探测、崩溃后退避重启本机所承担的服务—— 这套自愈逻辑三平台一致。
部署边界:本机(控制面)只托管 Host :8443; 数据面 Client :8444 运行在各终端用户机器上,由那台机器自己的守护托管 (终端机部署时设 CLAWWORKER_MANAGED_SERVICES=client),不在本机管理范围内。 {% if snap.platform == 'darwin' %} macOS 用 LaunchAgent(RunAtLoad+KeepAlive), 登录即启动,且守护自身被 launchd KeepAlive 兜底。 {% elif snap.platform == 'linux' %} Linux 用 systemd --user(Restart=on-failure); 如需"未登录也自启",已尝试 loginctl enable-linger。 {% elif snap.platform == 'windows' %} Windows 用计划任务(schtasks /SC ONLOGON),登录时启动守护。 {% endif %}
